# 关键 Prompt 清单

> **说明**：以下 Prompt 为任务完成后根据记忆整理的典型版本，非对话逐字记录。实际对话更长、更随意，包含了大量中间调整、追问和纠错。

---

## 一、用于任务拆解

### Prompt 1：把模糊需求拆成具体交付结构

**内容**：
```
帮我创建一个个人技术博客的静态网站项目，放到项目 docs/ 目录下。

要求：
1. 纯 HTML/CSS/JS，不依赖任何框架或构建工具
2. 响应式设计，桌面端和移动端都要好看
3. 支持暗色模式切换（用 CSS 变量实现，localStorage 记住用户选择）
4. 首页：hero 区 + 文章卡片列表，每张卡片有标题、日期、标签、摘要
5. 文章详情页：良好排版，支持代码块、表格、引用、标题层级
6. 设计风格：简洁、中性色、技术博客风格
7. 创建 3 篇示例文章（内容稍后我填充）
```

**作用**：把"我要做一个博客"这个模糊想法，拆成了 7 个具体文件和一个清晰的执行顺序。没有这个 Prompt，AI 会问"你想做什么样的博客"，来回几轮才能对齐；有了它，AI 直接开始创建文件。

**AI 输出质量**：⭐⭐⭐⭐⭐

### Prompt 2：为文章生成结构大纲

**内容**：
```
帮我为《Claude Code 入门与最佳实践》这篇文章设计大纲。
目标读者：有编程基础但没深入使用过 Claude Code 的开发者。
文章需要包含：
1. Claude Code 是什么（简短）
2. 我常用的工作流（重点）
3. 几个关键技巧（实操向）
4. 踩过的坑
5. 学习路径建议
6. 总结

不要写产品介绍式的文字，要实操、具体、有个人经验感。
```

**作用**：把我零散的"想写一篇 Claude Code 文章"的想法，变成了一个有逻辑的 6 段式大纲。AI 原大纲没有"踩过的坑"这一节，是我要求加上的——因为我真实的写作动机就是分享踩坑经历，而不是写产品介绍。

**AI 输出质量**：⭐⭐⭐⭐

## 二、用于生成初稿

### Prompt 3：基于自己的要点扩展成段落

**内容**：
```
基于以下要点，帮我扩展成 2-3 段流畅的文章段落：

要点：Claude Code 和 ChatGPT 的核心区别——ChatGPT 是"回答者"，你问它答；
Claude Code 是"执行者"，它能直接操作文件、执行命令。这个转变对开发效率的
影响是根本性的。

风格：技术博客，口语化但不随意，可以用类比帮助理解。
```

**作用**：文章的"骨架"和"核心观点"是我自己写的，AI 负责把骨头裹上肉——扩展成流畅的段落。这个分工模式让文章既有我的真实观点，又有 AI 的语言流畅度。

**AI 输出质量**：⭐⭐⭐⭐（删掉了一个不准确的"自动驾驶"类比）

---

### Prompt 4：生成项目文档结构

**内容**：
```
帮我生成这个项目的 README.md。要求：
- 参考任务一仓库 deep003-ai-moat 的 README 格式
- 需要包含：仓库导航表格、一句话定位、技术栈、交付物清单、前置任务完成情况
- 所有文件路径链接要可点击
- 用 Markdown 格式
```

**作用**：重复性的格式化工作交给 AI——表格对齐、链接生成、目录导航——这些不需要我的判断力，但手写很花时间。AI 生成后几乎零修改。

**AI 输出质量**：⭐⭐⭐⭐⭐

## 三、用于修改和检查

### Prompt 5：检查暗色模式配色问题

**内容**：
```
检查 style.css 中的暗色模式配色方案。暗色背景下的代码块和正文对比度够不够？
代码块背景在暗色模式下看起来和页面背景太接近了，帮我调整一下，让代码块更突出一些。
同时检查一下链接颜色在暗色背景下是否可读。
```

**作用**：AI 生成的第一版暗色模式代码块对比度不够。这个 Prompt 让 AI 自己检查自己的输出，找出视觉问题并修正。比我自己在浏览器里反复改色值快得多。

**AI 输出质量**：⭐⭐⭐⭐（初始方案偏蓝，额外要求调灰了两度）

---

### Prompt 6：优化首页文案

**内容**：
```
首页 hero 区的标题和副标题帮我优化一下。目前的版本太像模板了。
我想表达：我是一个 CS 学生，正在用 AI 工具提升开发效率，这个博客记录这个过程。
风格：不装、不矫情、有技术人的实在感。
```

**作用**：AI 的默认输出容易像模板（"欢迎来到我的博客"）。这个 Prompt 给了 AI 具体的风格约束和要表达的核心信息，让 AI 在约束内发挥，而不是接受默认方案。

**AI 输出**：
- 方案 A："AI 时代的开发者笔记"
- 方案 B："用 AI 写代码，用手写思考"
- 方案 C："从 L2 到 L3：一个 CS 学生的 AI 工具链实录"

**我的选择**：方案 B。简洁、有个性、表达了态度——AI 是工具，思考是人做的。

---

### Prompt 7：文章最终润色

**内容**：
```
帮我润色以下段落，要求：
1. 保持原意不变
2. 让句子更流畅，减少冗余
3. 保持口语化的技术博客风格
4. 不要添加任何新观点

[我写的原文段落]
```

**作用**：文章我写完后，让 AI 做最后一遍语言抛光。它是纯执行角色——不改意思、不加观点、只优化表达。这是 AI 做得最好的环节，几乎不需要人再改。

**AI 输出质量**：⭐⭐⭐⭐⭐

## Prompt 使用总结

| 维度 | 发现 |
|------|------|
| 最有效的 Prompt | 具体、有约束条件、有示例的（#1、#4） |
| 最需要迭代的 | 涉及审美判断的（#5、#6），通常 2-3 轮 |
| AI 独立完成最好的 | 代码生成、格式排版、段落润色 |
| 必须人工介入的 | 审美决策、个人经验内容、优先级判断 |
| 关键技巧 | 每次 Prompt 给出"不要做什么"和"要什么风格"，比说"做好看一点"有效得多 |
