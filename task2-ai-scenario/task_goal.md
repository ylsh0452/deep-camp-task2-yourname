# 任务目标

## 任务二：AI 真实场景应用实验

### DEEP 营官方要求

选择一个真实存在的场景，明确其中的具体问题，设计 AI 工具使用流程，完成最终交付物，并说明 AI 在这个过程中到底提升了什么、有哪些局限、哪些判断必须由你自己完成。

### 我的任务目标

**一句话**：使用 AI 工具链（Claude Code + DeepSeek V4 Pro）从零搭建个人技术博客，产出 3 篇 AI 辅助写作的原创技术文章，完整记录 AI 协作工作流并量化效果差异。

### 具体目标拆解

| 目标 | 衡量标准 | 完成情况 |
|------|---------|---------|
| 博客网站搭建 | 可访问的静态网站，响应式 + 暗色模式 | ✅ 5 页面，纯 HTML/CSS/JS |
| 技术文章产出 | 3 篇原创文章，有个人经验，非 AI 代写 | ✅ 共约 5000+ 字 |
| AI 工作流记录 | 分阶段记录，含完整 Prompt 和人机分工 | ✅ 见 ai_workflow.md + prompts.md |
| 效果对比 | 时间/质量/学习收获 3 维度量化 | ✅ 节省约 72-78% 时间 |
| 任务复盘 | 诚实反思收获与不足 | ✅ 见 reflection.md |
| 过程截图 | 覆盖关键环节 | ✅ 6 张截图 |
| GitHub 部署 | 公开可访问 | ✅ GitHub Pages |

### 为什么这个目标值得做

1. **真实痛点**：任务一路线图中写了「建立技术输出」但一直未启动
2. **可量化**：有明确的时间对比和质量对比数据
3. **可展示**：博客在线可访问，文章可阅读
4. **可复盘**：完整的 Prompt 记录和工作流文档
