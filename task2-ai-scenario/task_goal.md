# 任务目标

## 具体任务

**我要用 AI（Claude Code + DeepSeek V4 Pro）完成一个个人技术博客网站和 3 篇技术文章，部署到 GitHub Pages 并生成公开链接。**

### 交付物清单（可逐项验收）

| 序号 | 交付物 | 验收标准 |
|------|--------|---------|
| 1 | 博客首页 `index.html` | 包含 hero 区 + 3 张文章卡片，卡片显示标题、日期、标签、摘要 |
| 2 | 文章详情页 × 3 | 每篇支持标题层级、代码块、表格、引用块、链接 |
| 3 | 响应式布局 | 桌面端（1280px）和手机端（375px）均可正常阅读 |
| 4 | 暗色模式 | 点击切换按钮可在亮/暗主题之间切换，刷新后保持选择 |
| 5 | 文章 1：《Claude Code 入门与最佳实践》 | 不少于 2000 字，包含常用工作流、3 个关键技巧、3 个踩坑记录、4 阶段学习路径 |
| 6 | 文章 2：《DeepSeek V4 Pro 接入 Claude Code 配置指南》 | 不少于 1500 字，包含 API 代理原理说明、完整配置步骤、3 个踩坑记录、性价比对比表 |
| 7 | 文章 3：《Git 工作流入门：从零到 GitHub 绿点墙》 | 不少于 1500 字，包含 8 个核心命令速查表、3 个实用场景、commit message 规范 |
| 8 | GitHub Pages 部署 | 公开链接可访问，`curl` 返回 HTTP 200 |

### 技术约束

- 纯 HTML + CSS + JavaScript，不依赖任何框架或构建工具
- 所有文件放在仓库 `/docs` 目录，通过 GitHub Pages 直接部署
- 文章内容必须基于个人真实经验，空泛的 AI 生成内容不合格

## 为什么这个任务满足五条标准

| 标准 | 满足情况 |
|------|---------|
| **具体** | 8 项交付物都有明确的验收标准（字数、功能、文件路径） |
| **可执行** | 每项都是"创建一个文件"或"写完一篇文章"级别的原子任务 |
| **可展示** | 博客网站在线可访问，任何人打开链接就能看到全部交付物 |
| **可验收** | 验收标准列在表格里，逐项检查即知是否完成 |
| **能在本期完成** | 实际耗时约 4 小时，在一个下午内完成（2026-07-13 已完成） |
