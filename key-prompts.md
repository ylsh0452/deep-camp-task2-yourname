# 关键 Prompt 清单

本文档记录任务实施过程中使用的关键 Prompt，包括使用场景、完整 Prompt 文本、AI 输出质量和我的评价。

---

## Prompt #1：项目启动

**使用场景**：从零开始搭建博客项目，创建目录结构和初始文件

**完整 Prompt**：
```
帮我创建一个人技术博客的静态网站项目，放到 D:\wadfaf\deep003-tech-blog\deliverables\blog\ 目录下。

要求：
1. 纯 HTML/CSS/JS，不依赖任何框架或构建工具
2. 响应式设计，桌面端和移动端都要好看
3. 支持暗色模式切换（用 CSS 变量实现，localStorage 记住用户选择）
4. 首页：hero 区 + 文章卡片列表，每张卡片有标题、日期、标签、摘要
5. 文章详情页：良好排版，支持代码块、表格、引用、标题层级
6. 设计风格：简洁、中性色、技术博客风格
7. 创建 3 篇示例文章（内容稍后我填充）
```

**AI 输出质量**：⭐⭐⭐⭐⭐  
**评价**：AI 完整理解了需求，自行规划了文件创建顺序，CSS 变量系统设计合理，HTML 结构清晰。

---

## Prompt #2：CSS 暗色模式微调

**使用场景**：检查并优化暗色模式下的颜色对比度

**完整 Prompt**：
```
检查 style.css 中的暗色模式配色方案。暗色背景下的代码块和正文对比度够不够？
代码块背景在暗色模式下看起来和页面背景太接近了，帮我调整一下，让代码块更突出一些。
同时检查一下链接颜色在暗色背景下是否可读。
```

**AI 输出质量**：⭐⭐⭐⭐  
**评价**：AI 正确识别了对比度问题并调整了色值。但给出的初始方案偏蓝，我额外要求调灰了两度。

---

## Prompt #3：首页文案优化

**使用场景**：首页 hero 区的标题和副标题

**完整 Prompt**：
```
首页 hero 区的标题和副标题帮我优化一下。目前的版本太像模板了。
我想表达：我是一个 CS 学生，正在用 AI 工具提升开发效率，这个博客记录这个过程。
风格：不装、不矫情、有技术人的实在感。
```

**AI 输出**：
- 方案 A："AI 时代的开发者笔记"
- 方案 B："用 AI 写代码，用手写思考"
- 方案 C："从 L2 到 L3：一个 CS 学生的 AI 工具链实录"

**我的选择**：方案 B（"用 AI 写代码，用手写思考"）  
**理由**：简洁、有个性、表达了对 AI 工具的态度——AI 是工具，思考是人做的。

---

## Prompt #4：文章大纲生成（以文章 1 为例）

**使用场景**：生成《Claude Code 入门与最佳实践》的文章结构

**完整 Prompt**：
```
帮我为《Claude Code 入门与最佳实践》这篇文章设计大纲。
目标读者：有编程基础但没深入使用过 Claude Code 的开发者。
文章需要包含：
1. Claude Code 是什么（简短）
2. 我常用的工作流（重点）
3. 几个关键技巧（实操向）
4. 踩过的坑
5. 学习路径建议
6. 总结

不要写产品介绍式的文字，要实操、具体、有个人经验感。
```

**AI 输出质量**：⭐⭐⭐⭐  
**评价**：大纲结构合理，但部分小标题太像官方文档。我做了一些调整：把"核心功能概览"改成"我常用的工作流"，加了"踩过的坑"这一节（AI 原大纲没有）。

## Prompt #5：文章段落扩展

**使用场景**：扩展自己写的文章大纲中的某个要点

**完整 Prompt**：
```
基于以下要点，帮我扩展成 2-3 段流畅的文章段落：

要点：Claude Code 和 ChatGPT 的核心区别——ChatGPT 是"回答者"，你问它答；Claude Code 是"执行者"，它能直接操作文件、执行命令。这个转变对开发效率的影响是根本性的。

风格：技术博客，口语化但不随意，可以用类比帮助理解。
```

**AI 输出质量**：⭐⭐⭐⭐  
**评价**：扩展质量很好，语言流畅。但我删掉了一个不准确的类比（AI 把 Claude Code 比作"自动驾驶"，这个类比会误导读者以为不需要自己判断）。

---

## Prompt #6：文章润色

**使用场景**：自己写完后让 AI 做最终润色

**完整 Prompt**：
```
帮我润色以下段落，要求：
1. 保持原意不变
2. 让句子更流畅，减少冗余
3. 保持口语化的技术博客风格
4. 不要添加任何新观点

[我写的原文段落]
```

**AI 输出质量**：⭐⭐⭐⭐⭐  
**评价**：润色是 AI 最擅长的任务之一。它不会改变原意，但能让句子更紧凑、过渡更自然。这个环节 AI 几乎不需要人的干预。

## Prompt #7：文档格式化

**使用场景**：生成 README 和复盘文档的结构

**完整 Prompt**：
```
帮我生成这个项目的 README.md。要求：
- 参考任务一仓库 deep003-ai-moat 的 README 格式
- 需要包含：仓库导航表格、一句话定位、技术栈、交付物清单、前置任务完成情况
- 所有文件路径链接要可点击
- 用 Markdown 格式
```

**AI 输出质量**：⭐⭐⭐⭐⭐  
**评价**：格式化任务交给 AI 几乎零修改。AI 自动识别了目录中的所有文件并生成导航表格。

---

## Prompt 使用总结

| 维度 | 发现 |
|------|------|
| **最有效的 Prompt** | 具体、有约束条件、有示例的 Prompt（如 #1、#7） |
| **最需要迭代的 Prompt** | 涉及审美判断的 Prompt（如 #2、#3），通常需要 2-3 轮 |
| **AI 独立完成最好的** | 代码生成、格式排版、段落润色 |
| **必须人工介入的** | 审美决策、个人经验内容、优先级判断 |
| **关键技巧** | 每次 Prompt 都给出具体的"不要做什么"和"要什么风格"，比只说"做好看一点"有效得多 |
